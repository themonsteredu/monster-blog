# 더몬스터학원 블로그 자동화 — 전체 구상·계획 문서

## 1. 목표
주제 한 줄(또는 특강 내용)을 입력하면, 더몬스터학원 양식에 맞는 네이버 블로그 글 +
이미지 + 고정 썸네일을 만들고, 네이버에 임시저장까지 해주는 프로그램.
발행은 사람이 마지막에 확인 후 직접 누른다(안전장치).

## 2. 큰 그림 (흐름)
주제 입력 → ① 글 생성 → ② 이미지 생성 → ③ 썸네일 생성 → ④ 사람 확인
→ ⑤ 네이버 임시저장(로봇) → 사람이 발행

## 3. 구성 부품 (파일별 역할)
- app.py : 모든 걸 묶는 화면(설정/글/이미지/네이버 탭). 도구 = Streamlit
- generator.py : 글 생성. 도구 = 앤트로픽 Claude API (모델 claude-sonnet-4-6)
- images.py : 이미지 생성 + 일러스트/실사 전환. 도구 = OpenAI gpt-image-1
- thumbnail.py : 고정 배경 + 제목 글자로 매번 같은 썸네일. 도구 = Pillow
- naver_robot.py : 크롬을 자동 조종해 네이버에 임시저장. 도구 = Selenium
- 전용틀.md : 글 작성 규칙(말투/구조/학원 소개/위치/검색 규칙). 코드와 분리되어 있어
  이 파일만 고치면 글 스타일이 바뀐다.

## 4. 설정값 vs 입력값
- 설정값(한 번 정하면 계속): API 키 2개, 네이버 계정, 썸네일 배경/폰트, 학원 정보(전용틀 안)
- 입력값(글마다 바뀜): 글 종류, 주제, 핵심 키워드, 핵심 내용, 이미지 스타일

## 5. 만든 순서 (쉬운 것 → 어려운 것)
1단계 글 생성 → 2단계 이미지 → 3단계 썸네일 → 4단계 확인 화면 → 5단계 네이버 로봇 → 6단계 안정화
핵심: 5단계 로봇이 막혀도 1~4단계는 그대로 쓸 수 있어 헛수고가 없다.

## 6. 검색 노출(SEO) 설계 — 전용틀에 반영됨
- 제목 앞쪽에 핵심 키워드 / 본문에 자연스럽게 3~5번 + 동의어
- 큰 키워드 말고 구체적 롱테일(월 검색량 100~500)
- 1,500자, 모바일 한 문단 3~4줄(체류시간)
- 검색이 중요한 글은 AI 이미지만 쓰지 말고 얼굴 없는 원본 사진을 섞기
- 발행 후 네이버 서치어드바이저에 수집 요청

## 7. 정직한 한계·위험 (특히 5단계 로봇)
- 네이버는 공식 자동 발행 통로(API)가 없어, 로봇이 화면을 클릭하는 방식이라 깨지기 쉽다.
- 네이버가 화면을 바꾸면 선택자(CSS/XPath)를 수정해야 한다.
- 로그인 중 캡차/보안 인증이 뜨면 사람이 직접 풀어야 할 수 있다.
- 이미지를 정확한 위치에 끼워넣는 부분은 실제 화면을 보며 다듬어야 한다(코드에 TODO).
- 그래서 발행이 아니라 '임시저장'까지만 하도록 설계했다.

## 8. 여러 사람이 쓰게 하려면 (배포 계획)
- 서버에서 로봇을 돌리는 방식은 권장하지 않음(남의 네이버 비번 보관 = 보안·법적 위험).
- 선택한 방향: '각자 컴퓨터에 설치하는 프로그램'.
  - 본인 네이버 계정/키는 본인이 자기 컴퓨터에 입력 → settings.json 에 로컬 저장.
  - 사장님 학원용과 배포용은 '속 알맹이가 같은 프로그램'. 배포는 여기에 설치 포장만 씌우는 일.
- 새로 생기는 숙제: 설치 파일 만들기(윈도/맥 따로), 사용자 지원, 업데이트 배포.

## 9. 비용 (대략)
- 글 생성: 앤트로픽 API 사용량만큼 소액 과금
- 이미지: OpenAI gpt-image-1, 장당 소액(품질·크기에 따라 다름)
- 실제 단가는 각 콘솔의 요금 안내로 확인할 것(변동 가능)

## 10. 보안 메모
- 키·비밀번호는 settings.json(이 컴퓨터)에만 저장, 외부 전송 없음.
- 학생 사진 등 미성년자 이미지는 보호자 동의 없이 공개/업로드하지 않는다.
